Implementation roadmap

Look Again is designed as a focused Android XR prototype, not a broad museum-platform launch. The goal is to test whether a short perspective on the real artwork can help visitors begin looking before they know what to ask.

Build scope

Rapid prototype

Browser prototype now, native Android XR prototype next.

Measurable interaction

Test first-question clarity, phone comparison, and do-not-remove-glasses rate.

Source-grounded content

Use public-domain artworks and vetted source notes; AI routes perspectives, not invented interpretations.

Small enough to ship

The MVP avoids museum-wide navigation, heavy 3D reconstruction, and partnership dependency.

Android XR hardware validation

The XR value above only matters if the real artwork remains clear. The hardware test checks the few conditions that decide that.

The test defines the display conditions for keeping attention on the real work.

Validation focus

  • Artwork remains visible in dim rooms
  • Color and contrast stay close enough to the real work
  • The overlay stays peripheral and brief
  • The mark stays attached to the same visible detail
  • Viewing remains comfortable

Implementation and validation plan

The first build is deliberately narrow: can a short perspective on the real artwork help you begin looking better than audio or phone guidance?

  1. Start with 3–5 public-domain artworks.
  2. Build a native Android XR prototype.
  3. Select works by artwork number or QR code.
  4. Keep the MVP focused: no museum-wide navigation, no heavy 3D reconstruction, no museum partnership claim.
  5. Compare with audio and phone guidance before claiming XR is better.

Six-month roadmap

The six-month milestone is a closed-test MVP. The nine-month target is an Android XR release candidate.

  1. Month 1

    Native Android XR project setup.

    Port browser cue model into Android / Jetpack XR prototype.

  2. Month 2

    Artwork recognition / selection flow.

    Implement 3 public-domain artwork packs.

  3. Month 3

    Spatial cue rendering.

    Overlay, pulse, connector, peripheral panel, disappear-by-default behavior.

  4. Month 4

    Android XR hardware validation.

    Brightness, color fidelity, comfort, distraction, placement; include XREAL Project Aura if supplied through Catalyst.

  5. Month 5

    User study.

    Compare no guide / audio-like context / phone guide / Look Again cue.

  6. Month 6

    Closed test package.

    Demo video, Google Play closed testing readiness, public documentation.

Beyond museums

Museums are the first proof environment because the problem is visible there: people can be looking at the real thing and still not know how to enter the work.

The larger vision is a pre-search perspective layer for the physical world. Before a person knows what to ask, Android XR can reveal one meaningful detail, open one grounded question, and disappear.

Future contexts

Future directions may include cultural sites, architecture, and field learning. These are not part of the MVP.
